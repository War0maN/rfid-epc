# Шошго хэвлэх (Zebra RFID) — Төлөвлөгөө

> Энэ бол **хэлэлцэх төлөвлөгөө** — код хараахан бичээгүй. Доорх "Нээлттэй
> асуултууд"-ыг хамтдаа шийдсэний дараа хэрэгжүүлж эхэлнэ.

## 1. Зорилго

EPC хүснэгтээс **шүүж сонгосон** RFID-уудаа (хайрцгаар / SKU-гаар / нэг ширхэгээр)
аппаас **шууд Zebra принтер рүү** хэвлэх. Шошгоны загварыг тохируулж, хэвлэхээс
өмнө **урьдчилан харах** (preview).

## 2. Архитектур — Zebra Browser Print

Браузер аюулгүй байдлын улмаас вэб хуудас принтер рүү шууд холбогдож чаддаггүй.
Zebra үүнд зориулж **Browser Print** хэрэгслийг гаргасан:

```
[Вэб апп (браузер)]  →  HTTP (localhost)  →  [Zebra Browser Print]  →  USB/LAN  →  [Zebra принтер]
```

- Хэрэглэгч принтертэй компьютер дээр **Zebra Browser Print**-ийг **нэг удаа** суулгана
  (Windows/Mac, үнэгүй).
- Апп нь Browser Print-ийн JavaScript SDK ашиглан:
  - холбогдсон принтерүүдийг **олж жагсаах**,
  - сонгосон принтер рүү **raw ZPL** (RFID `^RFW` командтай) илгээх.
- RFID чип бичих бүрэн ажиллана (бид аль хэдийн зөв ZPL үүсгэдэг).

**Давуу тал:** BarTender/Zebra Designer-ийн "шууд хэвлэх"-тэй яг адил, нэмэлт
сервер шаардахгүй, RFID-д бүрэн нийцтэй.

**Анхаарах:** зөвхөн Browser Print суулгасан компьютероос хэвлэнэ (принтер тэр
компьютерт холбоотой). Утсаар/алсаас хэвлэх бол өөр шийдэл хэрэгтэй.

## 3. Хэрэглэгчийн урсгал

1. **EPC хүснэгт** дээр шүүнэ (ж: `Хайрцаг = CTA034752455`, эсвэл `SKU = IH6569-260`).
2. (Сонголт) тодорхой мөрүүдийг **checkbox**-оор сонгоно, эсвэл шүүсэн бүгдийг.
3. **"Хэвлэх"** товч дарна → принтер сонгох + тоо ширхэг батлах цонх.
4. Хэвлэхээс өмнө **preview** (нэг шошгоны харагдац) харагдана.
5. **Хэвлэх** → ZPL багц Browser Print-ээр принтер рүү очно.
6. Хэвлэгдсэн ажлыг тэмдэглэнэ (jobs.status = 'printed', аудит лог).

## 4. Шошгоны загвар (template)

### 4.1 Тохируулга (Тохиргоо хуудас / тенант тус бүр хадгална)
- **Хэмжээ:** өргөн × өндөр (мм). Ж: 50 × 30 мм, 40 × 25 мм.
- **DPI:** 203 (8 dot/mm) эсвэл 300 (12 dot/mm).
- **Талбарууд** (асаах/унтраах, байрлал):
  - Барааны нэр / SKU / GTIN
  - Баркод (1D: Code128 эсвэл EAN-13; эсвэл 2D: QR/DataMatrix)
  - Хүн уншихуйц EPC / Serial / Хайрцаг
- **Фонт хэмжээ**, баркодын өндөр.
- **RFID:** EPC-г чипэд бичих (`^RFW,H` — аль хэдийн бэлэн).

### 4.2 Preview
- ZPL-г зураг болгож харуулна. Хоёр сонголт:
  - **Labelary API** (labelary.com) — ZPL → PNG (хялбар, гэхдээ интернэт шаардлагатай).
  - **Өөрсдөө SVG render** (офлайн, гэхдээ ажил их).
- Эхэлж Labelary-аар хийгээд, шаардвал офлайн руу шилжиж болно.

### 4.3 Template хадгалалт
- `label_templates` хүснэгт (tenant_id, нэр, хэмжээ, талбарын тохиргоо JSON).
- Олон template хадгалж, хэвлэхдээ сонгоно (ж: "Хувцасны шошго", "Хайрцагны шошго").

## 5. Хэрэгжүүлэх үе шатууд

- **Phase 1 — Шууд хэвлэлт (хамгийн чухал):**
  Browser Print холболт + одоогийн ZPL-ээ шүүсэн мөрүүдээс шууд хэвлэх.
  (Загвар хатуу, гэхдээ ажиллана.)
- **Phase 2 — Сонголт + preview:**
  Мөр сонгох checkbox, нэг шошгоны preview (Labelary).
- **Phase 3 — Тохируулдаг template:**
  Хэмжээ/DPI/талбар/баркод сонгох тохиргоо + хадгалах.
- **Phase 4 (сонголт) — Бүрэн drag-drop дизайнер:**
  Хэрэв хэрэгтэй бол.

## 6. Нээлттэй асуултууд (хамтдаа шийдэх) ⚠️

1. **Принтерийн загвар:** Zebra-ийн ямар модель вэ (ж: ZD621R, ZT411R)? DPI хэд вэ (203/300)?
2. **Шошгоны биет хэмжээ:** өргөн × өндөр (мм)? RFID inlay-ийн байрлал?
3. **Шошгон дээр юу байх вэ:** нэр, SKU, GTIN баркод, хайрцаг, EPC… аль нь? Дараалал?
4. **Баркодын төрөл:** Code128 (SKU/GTIN), EAN-13 (баркод), эсвэл QR/DataMatrix?
5. **Кирилл текст:** барааны нэр кирилл үсгээр хэвлэх үү? (Тийм бол принтерт Unicode
   фонт ачаалах эсвэл текстийг зураг болгох хэрэгтэй.)
6. **Жишээ шошго:** одоо ашигладаг шошгоныхоо загвар (зураг/BarTender файл) байвал
   хуваалцаарай — яг тааруулна.
