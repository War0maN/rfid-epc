# Шошго хэвлэх — BarTender маягийн дизайнер (Zebra RFID)

> **Статус:** Архитектур шийдэгдсэн (доорх ✅). Код хараахан байхгүй — "Дараагийн
> алхамд хэрэгтэй мэдээлэл" хэсгийг бөглөсний дараа Phase 1-ээс эхэлнэ.

## 0. Шийдвэрүүд (баталгаажсан)

- ✅ **Хэлбэр:** Вэб апп дотор canvas дизайнер + **Zebra Browser Print** (тусдаа
  desktop програм биш).
- ✅ **Render:** **Bitmap / WYSIWYG** — зурсан шошгоо зураг болгож `^GFA`-аар
  хэвлэнэ. Кирилл, ямар ч фонт, зураг, эргүүлэлт, чөлөөт байрлал бүрэн дэмжинэ.
- ✅ **RFID:** EPC-г `^RFW,H`-ээр чипэд бичнэ (зурагнаас тусдаа).

## 1. Зорилго

BarTender/Zebra Designer шиг: принтер сонгох → шошгоны хэмжээ оруулах → **чөлөөтэй
зураглах** (текст, зураг/лого, баркод, RFID талбар; фонт солих, эргүүлэх) →
системийн **барааны дататай холбох** → шүүж сонгоод **шууд хэвлэх**.

## 2. Принтерүүд

- **Zebra RFID:** ZD621R (203/300 dpi), ZT411R, ZT611 (industrial, сүлжээ).
- Бүгд **ZPL** — нэг код бүгдэд ажиллана. DPI-г template-д тааруулна (203=8 dot/mm,
  300=12 dot/mm).
- Chainway CP30 — болих төлөвтэй тул тооцохгүй.

## 3. Архитектур (bitmap pipeline)

```
[Canvas дизайнер (Konva.js)]
        │  хэрэглэгч зураглана (текст/баркод/зураг/RFID)
        ▼
[Template хадгалах]  →  label_templates (хэмжээ + объектууд JSON)
        │
        │  Хэвлэхэд: сонгосон EPC мөр бүрд:
        ▼
[Дата орлуулах (data merge)]  →  объектын утга = тухайн мөрийн талбар
        ▼
[Принтерийн DPI-д яг тааруулж rasterize]  →  цагаан/хар 1-bit bitmap
        ▼
[ZPL үүсгэх]  →  ^XA ^RFW,H<EPC> ^GFA<bitmap> ^XZ   (мөр бүрд нэг)
        ▼
[Zebra Browser Print SDK]  →  принтер рүү илгээх
```

### Яагаад bitmap?
- **Зурснаараа яг хэвлэгдэнэ** — фонт/байрлал/эргүүлэлт ZPL-ийн хязгаарт баригдахгүй.
- **Кирилл бүрэн** — текстийг зураг болгодог тул ямар ч фонт ажиллана.
- Баркодыг **принтерийн DPI-д** яг зурвал хурц, уншигдах сайн хэвээр.

## 4. Технологийн сонголт

- **Canvas editor:** `react-konva` (Konva.js) — чирэх/хэмжээ/эргүүлэх бариул бэлэн.
- **Баркод:** `bwip-js` — Code128, EAN-13, QR, DataMatrix-г canvas-д принтерийн
  DPI-аар зурна.
- **Хэвлэх:** Zebra **Browser Print** JS SDK — принтер олох, `device.send(zpl)`.
- **ZPL bitmap:** canvas → 1-bit monochrome → `^GFA` (ачаалал багатай, шахаж болно).

## 5. Дизайнерийн объектууд

| Объект | Тохиргоо | Дата холболт |
|--------|----------|--------------|
| Текст | фонт, хэмжээ, бold/italic, өнгө(хар), эргүүлэлт, тэгшлэл | Статик эсвэл талбар |
| Баркод | төрөл (Code128/EAN/QR/DataMatrix), өндөр, HRI | Талбар (SKU/GTIN/EPC…) |
| Зураг/лого | upload, хэмжээ, байрлал | Статик |
| RFID | EPC-г чипэд бичих | EPC талбар (заавал) |
| Шугам/хүрээ | зузаан | — |

**Холбож болох талбарууд:** барааны нэр, SKU, GTIN, EPC (hex), serial, хайрцаг,
ажлын №, ирсэн огноо, нийлүүлэгч + чөлөөт текст.

## 6. Дата загвар

```sql
create table label_templates (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) default current_tenant_id(),
  name text not null,
  width_mm numeric not null,
  height_mm numeric not null,
  dpi int not null default 203,
  objects jsonb not null default '[]',  -- зурагласан объектуудын тодорхойлолт
  created_at timestamptz not null default now()
);
-- RLS: тенант тус бүр өөрийн template (бусад хүснэгтийн адил policy).
```

## 7. Хэвлэх урсгал

1. **EPC хүснэгт** дээр шүүх (хайрцаг/SKU/нэг ширхэг) → мөр сонгох (checkbox эсвэл бүгд).
2. **"Хэвлэх"** → template сонгох + принтер сонгох + тоо батлах.
3. **Preview** (нэг шошго) харах.
4. **Хэвлэх** → мөр бүрд data-merge → bitmap+`^RFW` → Browser Print руу багц.
5. Тэмдэглэх: jobs.status='printed', аудит лог.

## 8. Үе шатууд

- **Phase 1 — Дизайнер (core):** хэмжээ оруулах, текст/баркод/зураг/RFID нэмэх,
  фонт/эргүүлэлт/байрлал, дата холбох, template хадгалах. Canvas preview.
- **Phase 2 — Хэвлэх:** Browser Print холболт, сонгосон мөрүүдэд data-merge,
  bitmap→ZPL→хэвлэх. Багц хэвлэлт.
- **Phase 3 — Өнгөлгөө:** олон template, undo/redo, тэгшлэгч шугам, хувилах,
  алдаа боловсруулалт.

## 9. Тодорхойлсон шаардлага (баталгаажсан) ✅

| Зүйл | Утга |
|------|------|
| Шошгоны хэмжээ | 54×34мм (цаасан, antenna 50×30, Impinj M730 128-bit), ~65–70×30–35мм антиметал (≈1мм зузаан) |
| Дизайн | Янз бүр → тогтмол template биш, **чөлөөт дизайнер** |
| DPI | **300** (12 dot/mm) — template-д сонгож болдог байх (203 ч дэмжинэ) |
| Текст | Монгол **ба** Англи → bitmap render-ээр шийднэ |
| Баркод | EAN-13 (барааны баркод), Code128, **QR/DataMatrix (EPC уншуулахад)** |

### Баркодын зөвлөмж
- **24 тэмдэгт EPC-г 1D (Code128)-д хэвлэхэд урт/нягт болж уншигдахгүй.** Тиймээс
  EPC уншуулах зориулалтаар **QR эсвэл DataMatrix** ашиглана — найдвартай, жижиг.
- Барааны **EAN-13** (13 орон) → энгийн 1D баркод асуудалгүй.
- Дизайнерт дөрвүүлэнг (Code128/EAN-13/QR/DataMatrix) дэмжиж, хэрэглэгч сонгоно.

## 10. Дараагийн алхам

Шаардлага тодорхой болсон тул **Phase 1 (дизайнер)**-ийг `claude/label-printing`
branch дээр эхэлнэ. Эхний хэрэгжүүлэлт:
- `label_templates` хүснэгт + RLS (schema-д нэмнэ).
- `react-konva` + `bwip-js` суулгах.
- "Шошго" таб: template үүсгэх/сонгох, хэмжээ+DPI оруулах, canvas дээр
  текст/баркод/QR/зураг/RFID объект нэмж зураглах, дата холбох, хадгалах.
- Sample дататай preview.

Дараа нь **Phase 2:** Browser Print холболт + сонгосон EPC-үүдэд data-merge →
bitmap+`^RFW` хэвлэх.
