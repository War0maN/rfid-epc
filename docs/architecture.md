# Системийн архитектур

Chipmo Inventory — олон компанийн (multi-tenant) барааны ширхэгчилсэн бүртгэлийн систем. Энэ баримт нь ойлголт, өгөгдлийн бүтэц, тогтсон шийдвэрүүдийг нэгтгэнэ. (Хуучин `RFID-EPC-Generator-Spec.md` нь Phase 0 үеийн түүхэн спек.)

## 1. Ерөнхий зураг

```
Excel packing list ──импорт──► products + jobs ──генерац──► epc_codes (ширхэг бүр)
                                                              │
        Шошго хэвлэх (ZPL) ──► Хэвлээгүй → Идэвхтэй ──► Гүйлгээ (борлуулах/шилжүүлэх/бусад/буцаах)
                                                              │
                              epc_events (бүх өөрчлөлтийн түүх) + transactions (гүйлгээний бүртгэл)
                                                              │
                                        Үлдэгдэл · Тайлан · Аудит · EPC хайлт (timeline)
```

- **Тенант** — компани; бүх хүснэгт `tenant_id`-тай, RLS-ээр тусгаарлагдана.
- **Хэрэглэгч** — `admin` (бүрэн) / `operator` (салбар + эрхээр хязгаарлаж болно).

## 2. EPC кодлол

| Тохиолдол | Кодлол | Тайлбар |
|---|---|---|
| Баркодтой (GTIN) бараа | **SGTIN-96** | GS1 стандарт: company prefix + item ref + serial |
| Баркодгүй бараа | **GID-96** | Тенантын manager_number + object_class + serial |

Serial нь тенант × бараа бүрд **хэзээ ч давтагдахгүй, дараалсан** — `serial_counters` хүснэгт дээр атом RPC (`allocate_serials_bulk`)-ээр олгогдоно. Устгагдсан EPC-ийн serial дахин ашиглагдахгүй.

## 3. Каталог

- **Ангилал** — 3 түвшин (Үндсэн → Дэд → Барааны), заавал бүгдийг бөглөх албагүй.
- **Шинж чанар** — глобал жагсаалт (Өнгө, Размер...); утга нь `products.attributes` jsonb-д.
- **Бараа (master)** — нэр, SKU, GTIN, үнэ, ангилал, шинж чанарууд. Импортоос автоматаар үүсдэг (gtin эсвэл ext_key-ээр давхардалгүй upsert).

## 4. EPC-ийн төлөвийн мөчлөг

```
Хэвлээгүй (unprinted) ──хэвлэх──► Идэвхтэй (active) ──┬─борлуулах──► Борлуулсан (sold) ──┐
                                       ▲               ├─бусад──────► Бусад гүйлгээ (other)─┤─буцаалт─► Идэвхтэй
                                       │               └─шилжүүлэх──► Шилжүүлж буй ─хүлээн авах/цуцлах─► Идэвхтэй
```

- **Хэвлэх** үед `printed_at` + автоматаар Идэвхтэй болно.
- **Шилжүүлэг 2 алхамт**: илгээхэд Шилжүүлж буй (үлдэгдлээс гардаг); очих салбар нь хүлээн авахад тэнд Идэвхтэй болно; цуцлавал эх салбартаа буцна.
- "Шилжүүлж буй"-г гараар тохируулах/өөрчлөхийг DB trigger хориглоно (зөвхөн гүйлгээний RPC).
- Админ гараар төлөв солиж болно (тэмдэглэлтэй); Борлуулсан/Бусад болгох нь жинхэнэ гүйлгээ болж бүртгэгдэнэ.

## 5. Гүйлгээ (transactions)

| Төрөл | Үйлдэл | EPC төлөв |
|---|---|---|
| Борлуулсан (sale) | Идэвхтэйг зарах | active → sold |
| Шилжүүлэг (transfer) | Салбар хооронд (2 алхамт) | active → transferring → active (шинэ салбарт) |
| Бусад гүйлгээ (other) | Актлах, алга болсон г.м. | active → other |
| Буцаалт (return) | Зарагдсан/гарсан бараа буцаах | sold/other → active |

- Нэг гүйлгээ = нэг салбарын EPC-үүд; `transaction_items`-д ширхэг бүр **тухайн үеийн үнийн snapshot**-той.
- UI: скан→сагс урсгал (салбар сонгох → тохирох EPC жагсаалт → RFID уншигч/гараар скан эсвэл дарж нэмэх → нийт дүнтэй баталгаажуулах).
- Гүйлгээ **хэзээ ч устгагдахгүй** (DELETE policy байхгүй) — immutable ledger зарчим.
- Бүх бичилт `create_transaction`/`receive_transfer`/`cancel_transfer` атом RPC-ээр (security definer, дотроо тенант/төлөв/салбар/эрхийн шалгалттай).

## 6. EPC түүх (epc_events)

Append-only event log — `epc_codes`-ийн insert/update дээрх DB trigger автоматаар бичнэ (кодын аль ч зам алгасахгүй):

`created · printed · transfer_out · transfer_in · transfer_cancel · sold · other · returned · status_change`

Мөр бүрд: өмнөх/шинэ төлөв, өмнөх/шинэ салбар, холбогдох гүйлгээ (tx_id), тэмдэглэл (reason), хэн (actor), хэзээ. `(tenant_id, epc_id, created_at)` индекстэй тул сая мөрд ч нэг EPC-ийн түүх хурдан. UI: Бараа (EPC) → EPC дээр дарахад Хайлт руу үсэрч timeline харуулна.

## 7. Үлдэгдэл ба Тайлан

- **Үлдэгдэл** = Идэвхтэй EPC-ийн тоо, бараа × салбараар (`stock_by_branch` view). Матриц UI; Нийт/Нийт үнэ нь харагдаж буй салбаруудаар; тоон дээр дарж EPC жагсаалт.
- **Тайлан (Борлуулалт)** = `epc_events`-ийн 'sold' дээр DB-aggregate (`report_sales` RPC: өдөр×салбар×бараа×хэрэглэгч) — гүйлгээгээр ч, гараар Борлуулсан болгосон ч орно. Бүлэглэлт client-side (өдөр/сар/салбар/бараа/хэрэглэгч). Хязгаарлалт: буцаалт хасагддаггүй.

## 8. Эрхийн систем (2 давхарга)

Бүх хязгаарлалт **UI (нуух) + DB (RLS/RPC)** давхар. Админ үргэлж бүрэн; **тохиргоогүй хэрэглэгч бүрэн** (backward compatible).

- **Салбарын scoping** (`user_branches` + `has_branch_access()`): оноогдсон оператор зөвхөн өөрийн салбар(ууд)ын EPC/үлдэгдэл/гүйлгээ/түүхийг харж, үйлдэл хийнэ. Салбарын НЭРС бүгдэд нээлттэй (шилжүүлгийн очих салбар сонгоход). Ирж буй шилжүүлгийг хүлээн авагч салбар харна.
- **Нарийвчилсан эрх** (`user_permissions` + `has_perm()`): таб харах (9) + үйлдлийн (10) эрх — импорт, хэвлэх, борлуулах, шилжүүлэх, хүлээн авах, буцаах, бараа/ангилал/салбар засах. Хэрэглэгчид табын "Эрх" модалоос checkbox-оор.
- ⚠️ RLS policy-д функцийг мөр бүрд дуудахгүй — `(select has_perm(...))` InitPlan хэлбэрээр (гүйцэтгэл).

## 9. Устгалын зарчим (immutable history)

- Гүйлгээ, epc_events — хэзээ ч устгагдахгүй.
- EPC: зөвхөн **Хэвлээгүй/Идэвхтэй** устгаж болно (админ, баталгаажуулалттай); Борлуулсан/Шилжүүлж буй/Бусад trigger-ээр, **гүйлгээний түүхтэй** нь FK-ээр төлөв хамаагүй хамгаалагдана.
- Бараа: EPC-тэй бол устгагдахгүй (эхлээд Ажлыг устгаж хэвлээгүй EPC-г цэвэрлэнэ). Ангилал: бараатай бол устгагдахгүй. Салбар: EPC-тэй бол устгагдахгүй.
- Алдааны залруулга = сөрөг үйлдэл (буцаалт, цуцлалт, төлөв солих) — устгал биш.

## 10. Аудит

`audit_log`: хүснэгтийн өөрчлөлтүүд DB trigger-ээр (before/after jsonb), бизнес үйлдлүүд (хэвлэх, төлөв солих, устгах, export, генерац) гараар `logAuditEvent`-ээр. Бөөн EPC үйлдэлд meta-д бараагаар задаргаа + эхний 100 EPC hex. UI: мөр дарахад дэлгэрэнгүй модал (юуг, өмнө/дараа, түүхий JSON).

## 11. Гол хүснэгтүүд

`tenants · profiles · invites · user_branches · user_permissions · branches · categories · attribute_defs · products · serial_counters · object_class_counters · jobs · epc_codes · epc_events · transactions · transaction_items · label_templates · audit_log`

View: `epc_full · products_full · stock_by_branch` (бүгд security_invoker → RLS үйлчилнэ)

## 12. Үе шатын түүх (товч)

Phase 0 EPC/шошго/каталог → 1 Бүтээгдэхүүн master → 2 Салбар → 3 Төлөвийн мөчлөг → 4 Үлдэгдэл → 5 Гүйлгээ + EPC түүх → 6 Тайлан → 2b Салбар scoping → 2c Эрхийн систем → UX багц (таслал, аудит дэлгэрэнгүй, EPC устгах, Буцаалт). Хийгдээгүй: i18n (3 хэл), тооллого, deploy, имэйл урилга.
